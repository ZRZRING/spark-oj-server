# DESIGN.md

### 文档
文档可以是一个 md，html，pdf 格式的文件，可以作为题面，博客，公告的元素展示。

### 题目
关于题面：由于一个题目可以拥有多个题面（参考 vjudge 的多题面功能），题面可以是直接录入的 md 文本或者 pdf 文件，或者一个链接到比赛的完整题面（类似于 QOJ 或 GYM），我们需要对题面元素单独维护，不过我们只需要 1 对 n 的关系。

### 权限设计
可以通过设置自定义的 user_role 管理权限组，存储为 jsonb 格式，可以随业务扩充添加新的权限认证。

用户角色拥有 locked，user，admin，root 基础角色，不需要再去查 user_role 表。

- locked：无任何权限。
- user：具有前台用户权限。
- admin：具有进入后台权限。
- root：具有完全权限。
